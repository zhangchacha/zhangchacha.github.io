leetcode之4sum

发表于 2018-12-26 | 评论数： | 阅读次数：

今天开始刷LeetCode。因为之前在读研之前有段闲暇时间，所以有一部分简单的题目已经做过了。这次打算从前往后刷。

遇到的第一题是中等难度的4sum问题，即四数之和等于给定值的问题。

给定一个包含 n 个整数的数组 nums 和一个目标值 target，判断 nums 中是否存在四个元素 a，**b，c 和 d ，使得 a + b + c + d 的值与 target 相等？找出所有满足条件且不重复的四元组。

注意：

答案中不可以包含重复的四元组。

第一眼看上去没什么问题，直接暴力四重循环，判断和是不是等于给定的值：

不出意外的错了。检查一遍是没有判重。

那就判重嘛，多大点事，不过我觉得一个中等难度的不应该就这样子。于是加了判重：

就把最后的返回改掉了，但是还是错，报的错是

Line 17: TypeError: unhashable type: 'list'

搜了一下，是说元素为list的元素不能变成一个set，因为set是不可变的，但是list是可变的，所以不能哈希。这也简单呀，append的时候用tuple，然后返回的时候再改成list不就好了。这种小trick我最擅长了。

emmmm最担心的事情终于来了，时间超了。那怎么办呢，虽然刚开始的时候也知道时间最容易挂，但是真的挂了还是很烦。而且这种逻辑简单的还不容易优化时间。

于是想了个馊主意。能不能把四个数的和改成两个数的和，先把所有的数两两组合成两个数的和，再在这个基础上进行两数和的求解，不就是万能的二分思想吗，我真是个小机灵鬼。

这里的triple就是我的两个数的和，本来想用来存相加的两个数和他们的和，以便匹配的时候直接拿来append，但是后来发现还是需要去重，因为两个相加的部分可能数据一样但是位置不一样，也可能位置一样。比如这个例题，你在相加的两部分里分别都有0，你还是不能确定原始数组中是不是有两个0，无法去重。因此在三元组中又加了两个数在原始数组（其实是排序之后的原始数组）中的位置。这样逻辑就没问题了。

然而还是时间不够。分析了一下，原因是因为，虽然看起来像是二分，但是实际上的复杂度还是n^4量级，或许是3.5？反正就是复杂度还是很高。于是打算在第二次两数和的地方剪枝，但是剪枝这个东西写起来还很烦，而且个人觉得很low，就在想有没有优雅一些的剪枝方案。你还别说，真让我想到一个。就是在两数和的问题上，遍历其中一个数，另一个数就可以根据前一个数在很小的范围内移动。基本思想是这样，但是实现上出现了一些小问题，就是如果某次匹配的话，第二个指针要不要动呢？如果向前动的话下一次匹配就可能会带来数据丢失，不动的话会带来复杂度的增加而且也可能会带来数据丢失。写到一半越想越气，就没写下去。

后来想了一下应该是在无重复的数据上可以用。但是懒得去验证了。

最后还是决定在最初始的版本上进行剪枝。也没什么好说的，就是在最后一层先判断初始和末尾，如果第一个都大了，那么之后的肯定都大了，最后一个同理。同时，在计算过程中，如果大了，就直接进入下一个循环，因为之后的肯定都会大。

这里还有一个想法是，每次从中间开始遍历。可以根据数据的大小选择向两端计算。但是如果中间刚好匹配了，就不知道怎么办了。所以还是放弃了。

加了剪枝之后通过了，但是好像很差的样子：

执行用时: 1844 ms, 在4Sum的Python3提交中击败了1.72% 的用户

but，不管怎么样过了就行。也没有那么多需求啦。

听了室友的意见之后，好像可以用哈希表把存在判定的复杂度变为O（1）相应的整个暴力方法的复杂度就是n^3。懒得去验证了，爱怎么样就怎么样吧。
